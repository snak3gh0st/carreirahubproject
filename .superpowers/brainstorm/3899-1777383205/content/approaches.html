Abordagens para a Área de Gestão de Equipe

Escolha o padrão de UX que preferir para criar, editar e remover usuários

A

Tabela + Modal

Tudo em uma página. Botão "Novo Usuário" abre modal. Edição e desativação também via modal. Compacto, sem navegação.

Prós

  • Uma única rota
  • Sem perda de contexto

Contras

  • Modal pode ficar apertado
  • Difícil expandir depois
B

Tabela + Sheet Lateral ⭐ Recomendado

Lista de usuários principal. Clicar em "Novo" ou em um usuário abre um painel lateral (sheet) deslizante com o formulário. Segue o padrão já usado no projeto.

Prós

  • Contexto sempre visível
  • Espaço generoso para o form
  • Padrão moderno de admin

Contras

  • Levemente mais complexo de implementar
C

Lista + Páginas Separadas

Padrão CRUD clássico: /team lista usuários, /team/new cria, /team/[id]/edit edita. Simples e direto.

Prós

  • Mais simples de implementar
  • URLs diretas

Contras

  • Navegar para editar perde contexto da lista
