<task type="auto">
  <name>Task 2: Create comprehensive setup guide</name>
  <files>.planning/phases/05-docusign-production-setup/SETUP-GUIDE.md</files>
  <action>
Create detailed step-by-step production setup guide:

**Structure:**

1. **Overview**
   - What this setup accomplishes
   - Prerequisites (DocuSign production account, admin access, Vercel access)
   - Estimated time: 30 minutes

2. **Step 1: Generate RSA Keypair**
   - Run `npx ts-node scripts/docusign-setup.ts`
   - Save private.pem and public.pem files
   - ⚠️ Security warning: Keep private key secret, never commit to git

3. **Step 2: Create Integration Key in DocuSign**
   - Navigate to DocuSign Admin Console -> Apps and Keys
   - Click "ADD APP AND INTEGRATION KEY"
   - Name: "Carreira AI Hub Production"
   - Redirect URI: https://carreirausa.sigmaintel.io/api/auth/callback/docusign (or production URL)
   - Authentication: Enable "JWT Grant"
   - Click "ADD RSA KEYPAIR" and upload public.pem contents
   - Copy Integration Key (GUID) from result page

4. **Step 3: Get User ID and Account ID**
   - User ID: Admin Console -> Users -> Click on your user -> Copy "API Username" (GUID)
   - Account ID: Admin Console -> Account -> Copy "Account ID" (8-digit number)
   - Base URL: Check account region (e.g., NA4 -> https://na4.docusign.net)

5. **Step 4: Configure Vercel Environment Variables**
   - Navigate to Vercel dashboard -> Project Settings -> Environment Variables
   - Add each DOCUSIGN_* variable from script output:
     - DOCUSIGN_INTEGRATION_KEY
     - DOCUSIGN_USER_ID
     - DOCUSIGN_ACCOUNT_ID
     - DOCUSIGN_BASE_URL
     - DOCUSIGN_PRIVATE_KEY (entire PEM contents, preserve newlines)
     - DOCUSIGN_WEBHOOK_SECRET (generate random 32-char string)
     - DOCUSIGN_TEMPLATE_ID (optional, leave empty for now)
   - Scope: Production only
   - Redeploy after adding variables

6. **Step 5: Grant JWT Consent** ⚠️ CRITICAL
   - Open consent URL from script output
   - Login as DocuSign admin
   - Click "Allow" to grant permission
   - **This is one-time only** - JWT auth won't work until consent granted
   - You'll see "consent_granted" success page

7. **Step 6: Verify Setup**
   - Wait for Vercel deployment to complete
   - Proceed to Phase 05-02 for production testing

**Troubleshooting Section:**
- "consent_required" error: Admin hasn't granted consent via URL
- "invalid_grant" error: Integration Key or User ID incorrect
- "unauthorized_client" error: Public key not uploaded or mismatch with private key
- "account_not_found" error: Account ID incorrect or wrong base URL

**Security Best Practices:**
- Delete temporary keypair files after upload
- Store private key only in Vercel env vars (never in code)
- Rotate keypair annually
- Use different integration keys for sandbox vs production

Include screenshots or ASCII diagrams showing DocuSign console navigation.
