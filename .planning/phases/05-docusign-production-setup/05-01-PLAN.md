**Create `scripts/test-docusign-jwt.ts`:**
```typescript
import { docusignService } from '@/lib/services/docusign.service';

async function testJWTAuth() {
  console.log('🔐 Testing DocuSign JWT Authentication...\n');
  
  // Check if credentials are configured
  const integrationKey = process.env.DOCUSIGN_INTEGRATION_KEY;
  const userId = process.env.DOCUSIGN_USER_ID;
  const accountId = process.env.DOCUSIGN_ACCOUNT_ID;
  const privateKey = process.env.DOCUSIGN_PRIVATE_KEY;
  const baseUrl = process.env.DOCUSIGN_BASE_URL || 'https://demo.docusign.net';
  
  console.log('📋 Configuration Check:');
  console.log(`  Integration Key: ${integrationKey ? '✓ Set' : '✗ Missing'}`);
  console.log(`  User ID: ${userId ? '✓ Set' : '✗ Missing'}`);
  console.log(`  Account ID: ${accountId ? '✓ Set' : '✗ Missing'}`);
  console.log(`  Private Key: ${privateKey ? '✓ Set (' + privateKey.substring(0, 50) + '...)' : '✗ Missing'}`);
  console.log(`  Base URL: ${baseUrl}\n`);
  
  if (!integrationKey || !userId || !accountId || !privateKey) {
    console.log('❌ Missing required credentials. Please set:');
    console.log('   - DOCUSIGN_INTEGRATION_KEY');
    console.log('   - DOCUSIGN_USER_ID');
    console.log('   - DOCUSIGN_ACCOUNT_ID');
    console.log('   - DOCUSIGN_PRIVATE_KEY\n');
    
    if (integrationKey && userId) {
      const oauthBasePath = baseUrl.includes('demo')
        ? 'https://account-d.docusign.com'
        : 'https://account.docusign.com';
      
      const consentUrl = `${oauthBasePath}/oauth/auth?response_type=code&scope=signature%20impersonation&client_id=${integrationKey}&redirect_uri=https://www.docusign.com`;
      
      console.log('📝 Grant Consent URL (admin must visit once):');
      console.log(`   ${consentUrl}\n`);
      console.log('After granting consent, the admin will be redirected. The consent is persistent.');
    }
    
    process.exit(1);
  }
  
  // Generate consent URL
  const oauthBasePath = baseUrl.includes('demo')
    ? 'https://account-d.docusign.com'
    : 'https://account.docusign.com';
  
  const consentUrl = `${oauthBasePath}/oauth/auth?response_type=code&scope=signature%20impersonation&client_id=${integrationKey}&redirect_uri=https://www.docusign.com`;
  
  console.log('📝 Grant Consent URL (visit once as admin):');
  console.log(`   ${consentUrl}\n`);
  
  // Attempt JWT authentication
  try {
    console.log('🔑 Attempting JWT authentication...');
    const token = await docusignService.authenticateWithJWT();
    
    console.log('✅ JWT Authentication SUCCESSFUL!');
    console.log(`   Access Token: ${token.substring(0, 20)}...`);
    console.log(`   Token Length: ${token.length} characters\n`);
    
    console.log('✅ DocuSign production credentials are working correctly!');
    process.exit(0);
    
  } catch (error) {
    console.error('❌ JWT Authentication FAILED:');
    console.error('   ' + (error instanceof Error ? error.message : String(error)));
    
    if (error instanceof Error && error.message.includes('consent_required')) {
      console.log('\n📝 Action Required: Grant admin consent by visiting:');
      console.log(`   ${consentUrl}`);
      console.log('\nAfter granting consent, run this script again.');
    }
    
    process.exit(1);
  }
}

testJWTAuth();
```

<task type="checkpoint:human-action" gate="blocking">
  <what>DocuSign production account setup and credential configuration</what>
  <why>
DocuSign JWT authentication requires:
1. RSA keypair generated in DocuSign admin console (only admins can do this)
2. Integration Key (User Application) created
3. Admin consent granted via OAuth URL
4. Credentials configured in Vercel production environment

These actions require DocuSign account admin access and cannot be automated.
  </why>
  <instructions>
**Step 1: Access DocuSign Production Account**

Visit: https://account.docusign.com (or https://appdemo.docusign.com if using developer/demo account)

Login with admin credentials.

---

**Step 2: Create Integration Key and Generate RSA Keypair**

1. Navigate to: **Settings** -> **Integrations** -> **Apps and Keys**
2. Click: **Add App and Integration Key**
3. Enter App Name: "Carreira AI Hub Finance Integration"
4. Click: **Create App**
5. Under **Authentication**, click: **Generate RSA Keypair**
6. **IMPORTANT:** Download the private key file immediately (you cannot retrieve it later)
7. Copy the **Integration Key** (GUID format like `a1b2c3d4-...`)

---

**Step 3: Get User ID and Account ID**

1. Still in **Settings** -> **API and Keys**
2. Copy **User ID** (GUID format under "API Username")
3. Copy **Account ID** (under "API Account ID")

---

**Step 4: Note Base URL**

For production accounts, the base URL is typically:
- **NA**: `https://na4.docusign.net`
- **EU**: `https://eu.docusign.net`
- **Demo**: `https://demo.docusign.net`

Check your account's API endpoint in Settings -> API and Keys.

---

**Step 5: Run Test Script to Get Consent URL**

```bash
# Set credentials temporarily to generate consent URL
export DOCUSIGN_INTEGRATION_KEY="<your-integration-key>"
export DOCUSIGN_USER_ID="<your-user-id>"
export DOCUSIGN_ACCOUNT_ID="<your-account-id>"
export DOCUSIGN_PRIVATE_KEY="$(cat /path/to/downloaded/private.key)"
export DOCUSIGN_BASE_URL="https://na4.docusign.net"

# Run test script
npm run test:docusign-jwt
```

The script will output a consent URL like:
```
https://account.docusign.com/oauth/auth?response_type=code&scope=signature%20impersonation&client_id=<integration-key>&redirect_uri=https://www.docusign.com
```

---

**Step 6: Grant Admin Consent**

1. Copy the consent URL from the script output
2. Open the URL in your browser (must be logged in as admin)
3. Review the permissions request (signature + impersonation scopes)
4. Click **Allow Access** / **Grant Consent**
5. You'll be redirected to docusign.com (this is expected - consent is granted)

**Note:** This consent is permanent - you only need to do it once per Integration Key.

---

**Step 7: Configure Vercel Production Environment Variables**

Visit: https://vercel.com/your-team/carreirausa/settings/environment-variables

Add the following environment variables:

| Variable | Value | Source |
|----------|-------|--------|
| `DOCUSIGN_INTEGRATION_KEY` | `<integration-key-guid>` | Step 2 |
| `DOCUSIGN_USER_ID` | `<user-id-guid>` | Step 3 |
| `DOCUSIGN_ACCOUNT_ID` | `<account-id>` | Step 3 |
| `DOCUSIGN_PRIVATE_KEY` | `<private-key-contents>` | Downloaded file from Step 2 |
| `DOCUSIGN_BASE_URL` | `https://na4.docusign.net` | Step 4 |
| `DOCUSIGN_WEBHOOK_SECRET` | `<generate-random-string>` | Generate: `openssl rand -hex 32` |

**For DOCUSIGN_PRIVATE_KEY:**
- Option 1: Copy entire private key including `-----BEGIN RSA PRIVATE KEY-----` and `-----END RSA PRIVATE KEY-----`
- Option 2: Base64 encode: `cat private.key | base64`

**Redeploy** after adding env vars for changes to take effect.

---

**Step 8: Test Production Authentication**

After Vercel redeploy completes:

```bash
# SSH into Vercel production or use Vercel CLI
vercel env pull .env.production

# Test authentication
npm run test:docusign-jwt
```

Should output: `✅ JWT Authentication SUCCESSFUL!`

---

**Step 9: Configure DocuSign Connect for Webhooks (if not already done)**

1. Navigate to: **Settings** -> **Connect** -> **Add Configuration**
2. Configuration Name: "Carreira AI Hub Production"
3. Webhook URL: `https://carreirausa.sigmaintel.io/api/webhooks/docusign`
4. Events to send:
   - Envelope Sent
   - Envelope Delivered
   - Envelope Completed
   - Envelope Declined
   - Envelope Voided
5. **Include HMAC Signature**: Enabled
6. **HMAC Secret**: Use value from `DOCUSIGN_WEBHOOK_SECRET` env var (Step 7)
7. Save configuration

---

**Troubleshooting:**

**Error: "consent_required"**
- Solution: Grant consent via URL from Step 6

**Error: "invalid_grant" or "unauthorized_client"**
- Check Integration Key is correct
- Verify consent was granted
- Ensure User ID matches the account

**Error: "RSA key format invalid"**
- Private key must start with `-----BEGIN RSA PRIVATE KEY-----`
- Try base64 encoding if multi-line format causes issues

**Error: "Account ID not found"**
- Verify Account ID is from production account (not demo if using production)
- Check Base URL matches account type
