<objective>
Implement robust cron-based queue processing for all 9 BullMQ queues to replace workers (which don't run on Vercel) with a reliable polling strategy that processes jobs within serverless constraints.

Purpose: Vercel serverless functions don't support persistent worker processes. BullMQ's built-in workers cannot run. Solution: implement a cron job that polls Redis queues and processes jobs within 10-second timeout constraints, with per-queue processing limits to prevent timeout overruns.

Output: `/api/cron/process-queue` endpoint that safely processes all 9 queue types (leadQualification, whatsappMessages, pipedriveSync, invoiceGeneration, contractGeneration, quickbooksSync, pipedriveReverseSync, invoiceApproval, bulkImport) with job timeout handling and monitoring.
</objective>
