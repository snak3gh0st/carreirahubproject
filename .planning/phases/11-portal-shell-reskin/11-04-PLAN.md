<!-- Pattern 2 from research: GOLD Constant Elimination -->

Decision rule for each GOLD usage:

| Current usage | Replace with | Reason |
|---------------|-------------|--------|
| style={{ backgroundColor: GOLD }} on static element | className="bg-brand-tangerina" | Static — Tailwind class preferred |
| style={{ color: GOLD }} on static text/icon | className="text-brand-tangerina" | Static — Tailwind, but ONLY on dark bg |
| style={{ borderColor: GOLD }} | className="border-brand-tangerina" | Static border |
| style={{ backgroundColor: GOLD }} in ternary condition | BRAND_COLORS.TANGERINA | Dynamic/conditional — JS constant required |
| onFocus={(e) => e.target.style.borderColor = GOLD} | BRAND_COLORS.TANGERINA | Dynamic event handler — JS constant |
| accent-[#C9A84C] on radio/checkbox | style={{ accentColor: BRAND_COLORS.TANGERINA }} | CSS accent-color cant use vars in all browsers |
| style={{ backgroundColor: "#FBF8F0" }} | className="bg-brand-creme" | Token class |
| style={{ backgroundColor: "#FFF8E7" }} | className="bg-brand-creme" | Close enough to token |
| Text on Creme/white background | text-brand-verde (NOT text-brand-tangerina) | WCAG AA constraint |

From lib/constants/brand.ts:
```typescript

// BRAND_COLORS.TANGERINA = "#FF8142"

```

3. Replace each usage per the decision rule table above
