<objective>
Implement circuit breaker pattern for all external API calls to prevent cascading failures when integrations are temporarily unavailable.

Purpose: When external APIs (Pipedrive, QuickBooks, Stripe, DocuSign, Twilio, RetellAI, OpenAI, Resend) fail, the system should fail gracefully instead of repeatedly retrying and exhausting resources. Circuit breaker pattern prevents cascading failures by "opening" when error rates exceed threshold, returning pre-determined fallback responses instead of making external calls.

Output: Circuit breaker wrapper for all 8 external integrations with configurable thresholds, metrics tracking, and automatic recovery detection.
</objective>

Integrate CircuitBreaker into each service's main API call methods:

For each of the 8 services (Pipedrive, QuickBooks, Stripe, DocuSign, Twilio/WhatsApp, RetellAI, OpenAI, Resend):

1. Import CircuitBreaker at top of service file
2. Create singleton circuit breaker instance: `const cb = new CircuitBreaker("serviceName")`
3. Wrap each external API call method with circuit breaker:
   - Replace direct axios/fetch calls with cb.execute(() => axiosCall())
   - Return fallback response when circuit opens (see fallback strategy below)
   - Log circuit state changes to IntegrationLog with service, action, status, error fields

Fallback strategies per service:
- **Pipedrive** (CRM): Return cached customer/deal data if available, otherwise return null. Queue for retry via webhook.
- **QuickBooks** (Accounting): Return null/error response. Log to IntegrationLog for manual sync later.
- **Stripe** (Payments): Return error response to client (user can retry). Don't create invoice yet.
- **DocuSign** (Contracts): Queue contract for retry (async job). Return "pending" status.
- **Twilio/WhatsApp** (Messaging): Queue message for retry. Return status "queued".
- **RetellAI** (Voice): Queue call for retry. Return status "pending".
- **OpenAI** (Chatbot): Return fallback response ("We're experiencing high load..."). Queue for retry.
- **Resend** (Email): Queue email for retry. Return status "queued".

Key implementation pattern:
```typescript
try {
  return await cb.execute(async () => {
    return await makeExternalCall();
  });
} catch (error) {
  if (error instanceof CircuitOpenError) {
    // Circuit is open - return fallback
    return fallbackResponse;
  }
  // Other errors - propagate
  throw error;
}
```

DO NOT change service method signatures (preserve backward compatibility).
DO NOT add circuit breaker initialization to constructor (singleton pattern avoids repeated DB queries).
WHY: Backward compatible updates ensure existing code keeps working. Singleton pattern prevents flooding database with repeated state loads.
