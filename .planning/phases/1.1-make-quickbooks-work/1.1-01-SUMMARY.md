## Technical Decisions

### Decision 1: bcrypt → bcryptjs
**Context**: Native bcrypt module failed in Vercel serverless environment

**Options Considered**:
1. Fix bcrypt native build (compile for Lambda)
2. Use bcryptjs (pure JavaScript)
3. Use alternative hashing (argon2, scrypt)

**Decision**: Use bcryptjs
**Rationale**:
- Drop-in replacement (same API)
- No native compilation needed
- Battle-tested, widely used
- Slightly slower but acceptable for authentication
- No breaking changes to existing hashes

### Decision 2: Increase Invoice Limit to 5000
**Context**: Some QB accounts have >1000 invoices

**Decision**: Set default maxResults to 5000
**Rationale**:
- Handles most small-medium businesses
- Still within Vercel timeout limits (~10s)
- Pagination prevents memory issues
- User can adjust if needed

### Decision 3: Keep Debug Endpoints in Production
**Context**: Debug endpoints expose some internal info

**Decision**: Keep them (no auth required) for now, remove later
**Rationale**:
- Critical for diagnosing production issues
- Can be removed after system is stable
- No sensitive data exposed (passwords are hashed)
- Faster troubleshooting

## Issues Encountered

### Issue 1: bcrypt Native Module Failure
**Symptom**: Login failed with "Invalid password" even with correct credentials
**Root Cause**: bcrypt native binary not available in Vercel Linux environment
**Debug Steps**:
1. Created test endpoint to isolate issue
2. Checked database - hash was correct
3. Tested bcrypt.compare() - failed with module error
4. Identified: "No native build was found for platform=linux"
**Resolution**: Replaced with bcryptjs
**Time Lost**: ~1 hour of debugging

### Issue 2: Local vs Production Database Confusion
**Symptom**: Password worked locally but not in production
**Root Cause**: Local scripts updated local database, Vercel uses Neon production DB
**Resolution**: Created production-specific reset script using POSTGRES_PRISMA_URL
**Time Lost**: ~30 minutes

### Issue 3: JSON Escaping in curl Commands
**Symptom**: Test API calls failed with "Bad escaped character" errors
**Root Cause**: Shell escaping issues with special characters in passwords
**Resolution**: Used JSON files instead of inline JSON strings
**Learning**: Always use files or proper quoting for API testing

## Lessons Learned

1. **Always test in production-like environment**: bcrypt worked locally (macOS) but failed on Vercel (Linux)
2. **Pure JS > Native modules for serverless**: bcryptjs is more reliable than bcrypt in Lambda/Vercel
3. **Debug endpoints are invaluable**: Saved hours of troubleshooting
4. **Local vs production databases**: Easy to forget which DB you're updating
5. **Simple passwords for initial setup**: Complex passwords with special chars caused testing issues
