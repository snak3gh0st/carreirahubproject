  <name>Task 1: Create webhook health check API endpoint</name>
  <files>app/api/webhooks/health/route.ts, lib/utils/webhook-health.ts</files>
  <action>
Create app/api/webhooks/health/route.ts with GET endpoint that returns:
```json
{
  "status": "healthy" | "degraded" | "unhealthy",
  "timestamp": "2026-01-10T12:00:00Z",
  "services": {
    "pipedrive": { "status": "healthy", "successRate": 0.98, "recentErrors": 2 },
    "quickbooks": { "status": "healthy", "successRate": 1.0, "recentErrors": 0 },
    "stripe": { "status": "healthy", "successRate": 0.99, "recentErrors": 1 },
    "docusign": { "status": "degraded", "successRate": 0.75, "recentErrors": 15 },
    "twilio": { "status": "healthy", "successRate": 1.0, "recentErrors": 0 },
    "retell": { "status": "healthy", "successRate": 0.95, "recentErrors": 3 }
  },
  "deadLetterCount": 12,
  "pendingRetries": 8
}
```

Status calculation:
- "healthy": successRate >= 0.95 for all services
- "degraded": at least one service has 0.80 <= successRate < 0.95
- "unhealthy": any service has successRate < 0.80

Create lib/utils/webhook-health.ts with:
- calculateWebhookHealth(): Promise<HealthCheckResponse>
  - Query WebhookEvent table for last 24 hours
  - Calculate per-service success rates (success / total)
  - Count recent errors (status="failed", created_at > 24h ago)
  - Count dead letter queue (status="dead_letter")
  - Count pending retries (status="failed", next_retry_at < now)

This endpoint is PUBLIC (no auth required) for uptime monitoring tools like UptimeRobot to ping.

Use existing patterns:
- Import prisma from @/lib/db
- Use Prisma aggregations: groupBy, count, where clauses
- Return NextResponse.json()
