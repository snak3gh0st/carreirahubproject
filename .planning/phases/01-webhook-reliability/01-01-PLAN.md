<objective>
Implement webhook retry logic with exponential backoff and dead letter queue to prevent webhook event loss when processing fails.

Purpose: Ensure zero lost webhook events from Pipedrive, QuickBooks, Stripe, DocuSign, Twilio, and RetellAI by automatically retrying failed processing with exponential backoff and capturing permanently failed events in a dead letter queue for manual intervention.

Output: Webhook event persistence table, retry middleware, dead letter queue, and reprocess API endpoint.
</objective>

- 8 webhook endpoints (Pipedrive x3, QuickBooks, Stripe, DocuSign, RetellAI, Twilio)
- Signature validation exists (HMAC SHA256)
- Integration logging exists (IntegrationLog table)
- NO retry mechanism - failed webhooks are lost
- NO dead letter queue - no way to recover from permanent failures

- Vercel serverless (10s timeout per function)
- Must not block webhook response (return 200 quickly)
- Cannot use long-running workers (need async processing)
- Existing webhooks must continue working (backward compatible)

<task type="auto">
  <name>Task 2: Implement webhook retry middleware with exponential backoff</name>
  <files>lib/middleware/webhook-retry.ts, lib/utils/retry-logic.ts</files>
  <action>
Create webhook-retry.ts with handleWebhookWithRetry() wrapper function that:
1. Persists incoming webhook to WebhookEvent table (status="pending")
2. Attempts to process webhook via provided handler function
3. On success: updates status="success", sets processed_at
4. On failure: increments retry_count, sets next_retry_at using exponential backoff formula (2^retry_count * base_delay_ms), updates status="failed"
5. If retry_count >= max_retries: sets status="dead_letter" (permanent failure)
6. Always returns 200 OK to webhook sender (don't block their retry logic)
7. Logs all operations to IntegrationLog table

Create retry-logic.ts with:
- calculateNextRetryAt(retryCount: number, baseDelayMs: number = 60000): Date // 1min, 2min, 4min, 8min, 16min
- shouldRetry(retryCount: number, maxRetries: number): boolean

Use existing patterns:
- Import prisma from @/lib/db
- Import integrationLogger from @/lib/utils/logger
- Follow existing error handling patterns (try/catch, log errors)

DO NOT use any long-running background processes (Vercel constraint). Retries will be triggered by a cron job (implemented in next phase).
  </action>
  <verify>TypeScript compiles without errors, handleWebhookWithRetry function signature accepts (service, eventType, payload, headers, handler), retry logic calculates backoff correctly</verify>
  <done>Middleware persists webhooks, handles success/failure correctly, calculates exponential backoff, returns 200 OK immediately, logs to IntegrationLog</done>
</task>
