**Tech stack available:**
- Next.js 14 App Router
- PostgreSQL with Prisma ORM
- Tailwind CSS for styling
- TypeScript 5.x
- Existing dashboard at /dashboard with navigation

**Established patterns:**
- Service layer for business logic (lib/services/)
- API routes for data fetching (app/api/)
- Dashboard pages at app/dashboard/
- Force dynamic rendering for authenticated pages
- Mobile-responsive design (Phase 4.1)

**Constraining decisions:**
- Use Recharts (simple API, sufficient for Finance charts)
- Client-side caching with React Query (5min staleTime)
- Disable server caching (force-dynamic) for authenticated dashboard
- Lazy load charts to reduce initial bundle size

**Key files:**
- app/dashboard/layout.tsx - Dashboard navigation
- lib/services/quickbooks.service.ts - QuickBooks data access
- prisma/schema.prisma - Customer, Invoice, Payment, Deal models
