<task type="auto">
  <name>Task 1: Create API route for BI analytics data</name>
  <files>app/api/analytics/financial/route.ts</files>
  <action>
Create GET endpoint that aggregates financial data for the dashboard:

**Response structure:**
```typescript
{
  kpis: {
    totalRevenue: number,
    overdueAmount: number,
    collectionRate: number, // percentage
    activeCustomers: number
  },
  invoiceStatusDistribution: Array<{ status: string, count: number, value: number }>,
  revenueTrend: Array<{ month: string, revenue: number }>, // last 12 months
  topCustomers: Array<{ name: string, email: string, totalPaid: number }> // top 10
}
```

**Implementation:**
1. Use Prisma to aggregate data from Invoice, Payment, Customer tables
2. Calculate KPIs:
   - totalRevenue: `sum(amountPaid)` where status=PAID
   - overdueAmount: `sum(amount)` where status=OVERDUE
   - collectionRate: `(totalPaid / totalInvoiced) * 100`
   - activeCustomers: count distinct customers with invoices in last 90 days

3. Invoice status distribution: `groupBy status`, count and sum amounts

4. Revenue trend: Group payments by month (last 12 months), sum amount per month

5. Top customers: Join Customer + Payment, group by customer, order by total desc, limit 10

**Authentication:**
- Require authenticated user (check NextAuth session)
- Finance role preferred, but allow all authenticated users

**Performance:**
- Single database query per aggregation (no N+1 queries)
- Use Prisma aggregations and groupBy where possible
- Target <500ms response time

**What to avoid:**
- Do NOT create separate endpoints for each metric (one endpoint for all analytics)
- Do NOT fetch all invoices/payments then aggregate in JS (use SQL aggregation)
- Do NOT cache results server-side (dashboard uses force-dynamic)
