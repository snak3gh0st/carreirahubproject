<task type="auto">
  <name>Task 1: Add date range filters to BI dashboard</name>
  <files>app/dashboard/insights/page.tsx, app/api/analytics/financial/route.ts, components/dashboard/date-range-filter.tsx</files>
  <action>
**1. Create date range filter component** (components/dashboard/date-range-filter.tsx):
- Quick filter chips: Last 7 Days, Last 30 Days, Last 90 Days, This Year, All Time
- Custom date range picker (start date, end date) using HTML date inputs
- Apply button to submit custom range
- Mobile: horizontal scroll for chips (same pattern as Phase 1.1)
- Store selected filter in URL query params: ?dateRange=last30 or ?from=2025-01-01&to=2025-12-31

**2. Update API route** (app/api/analytics/financial/route.ts):
- Accept dateRange query param: last7, last30, last90, thisYear, allTime, custom
- Accept from and to query params for custom range (ISO date strings)
- Filter Invoice, Payment queries by date:
  - Invoices: filter by createdAt or paidAt (depending on metric)
  - Payments: filter by paymentDate
  - Default: All time (no filter)

**3. Update insights page** (app/dashboard/insights/page.tsx):
- Add DateRangeFilter component above KPI cards
- Read date range from URL query params
- Pass date range to React Query: `queryKey: ['analytics', dateRange, from, to]`
- Re-fetch data when date range changes
- Persist filter selection in URL (use Next.js useRouter and searchParams)

**Date calculations:**
- Last 7 Days: `subDays(new Date(), 7)` to `new Date()`
- Last 30 Days: `subDays(new Date(), 30)` to `new Date()`
- Last 90 Days: `subDays(new Date(), 90)` to `new Date()`
- This Year: `startOfYear(new Date())` to `new Date()`
- All Time: no filter
- Use date-fns: `subDays`, `startOfYear`, `format`

**What to avoid:**
- Do NOT filter data client-side (filter in API route via SQL)
- Do NOT use date pickers that require additional libraries (use native HTML date input)
- Do NOT break mobile responsiveness (chips should scroll horizontally)
