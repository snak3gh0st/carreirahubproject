## Decisions Made

1. **HTML5 native date inputs**: Used `<input type="date">` instead of JavaScript date picker libraries to avoid bundle bloat and maintain simplicity
2. **In-memory filtering for complex queries**: Applied invoice count and overdue balance filters in memory after database fetch (acceptable performance at current scale, can optimize later if needed)
3. **URL-based filter state**: All filters persist in URL query params enabling shareability, bookmarking, and deep linking
4. **Expandable advanced filters**: Used `<details>` element to hide advanced filters by default, reducing UI clutter while keeping them accessible
