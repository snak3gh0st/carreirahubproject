<task type="auto">
  <name>Task 2: Add advanced filtering UI to customer list page</name>
  <files>app/dashboard/customers/page.tsx</files>
  <action>
  Enhance customer list page with advanced filtering similar to invoice page.

  **New Filters to Add:**
  1. **Balance Status Filter:**
     - Dropdown: All / No Balance / Has Balance / Overdue Balance
     - Logic:
       - No Balance: `qbBalance = 0` OR `qbBalance IS NULL`
       - Has Balance: `qbBalance > 0`
       - Overdue Balance: Has unpaid invoices where dueDate < today
     - Note: "Overdue Balance" requires join with invoices table

  2. **Invoice Count Filter:**
     - "Min Invoices" - number input
     - "Max Invoices" - number input
     - Requires: Count invoices per customer, filter based on count
     - Implementation: Use Prisma `where: { invoices: { some: {} } }` (basic) or raw query for exact counts

  3. **Total Invoiced Range:**
     - "Min Total Invoiced" - currency input
     - "Max Total Invoiced" - currency input
     - Filter on `qbTotalInvoiced` field (synced from QuickBooks)

  4. **Created Date Range:**
     - "Created From" - date input
     - "Created To" - date input
     - Filter: `where: { createdAt: { gte: fromDate, lte: toDate } }`

  **Filter Bar Layout:**
  - Similar to invoice page: expandable "Advanced Filters"
  - When expanded: 2-column grid with filter controls
  - Apply and Clear buttons
  - Active filter count badge

  **Implementation:**
  - Parse searchParams: `balanceStatus`, `minInvoices`, `maxInvoices`, `minTotalInvoiced`, `maxTotalInvoiced`, `createdFrom`, `createdTo`
  - Build Prisma where clause dynamically
  - For "Overdue Balance": May need subquery or separate count (prioritize performance)
  - Simple approach for invoice count: Filter in JavaScript after fetch (acceptable for current scale)

  **What to avoid and WHY:**
