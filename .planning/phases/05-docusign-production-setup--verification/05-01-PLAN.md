<task type="checkpoint:human-action" gate="blocking">
  <action>Generate RSA keypair and create Integration Key in DocuSign production account</action>
  <instructions>
**IMPORTANT:** This task requires direct access to DocuSign Admin Console and cannot be automated.

**Step 1: Access DocuSign Production Admin Console**
1. Log in to https://admindemo.docusign.com or https://admin.docusign.com (production)
2. Navigate to **Settings** → **Apps and Keys**
3. Verify you have admin-level access

**Step 2: Create Integration Key Application**
1. Click **Add App and Integration Key**
2. Fill in application details:
   - App Name: "Carreira AI Hub Production"
   - Description: "Finance workflow automation - QuickBooks to DocuSign integration"
3. Click **Create App**
4. **Copy Integration Key** (GUID format) - save to secure note for Vercel env vars

**Step 3: Generate RSA Keypair**
1. In your new app, click **Actions** → **Generate RSA Keypair**
2. DocuSign generates 2048-bit RSA keypair
3. **Download Private Key** - save securely (never commit to git)
4. Public key is automatically registered with DocuSign
5. **Important:** Private key format options:
   - Multi-line PEM format (-----BEGIN RSA PRIVATE KEY-----)
   - Base64 encoded single line
   - Either format works for DOCUSIGN_PRIVATE_KEY env var

**Step 4: Get User ID and Account ID**
1. Navigate to **My Account** → **API and Keys**
2. **Copy User ID** (GUID format) - this is your DOCUSIGN_USER_ID
3. Navigate to **Settings** → **API and Keys**
4. **Copy Account ID** - this is your DOCUSIGN_ACCOUNT_ID

**Step 5: Determine Base URL**
1. Check your DocuSign account region:
   - Look at browser URL: `https://appXXX.docusign.com`
   - Base URL pattern: `https://naX.docusign.net` or `https://demo.docusign.net`
2. Common production base URLs:
   - North America: `https://na4.docusign.net`
   - Demo/Sandbox: `https://demo.docusign.net`
3. **Verify** by checking account settings or contacting DocuSign support if unsure

**Step 6: Grant Admin Consent for JWT**
1. Construct consent URL (replace placeholders):
   ```
   https://account.docusign.com/oauth/auth?response_type=code&scope=signature%20impersonation&client_id={YOUR_INTEGRATION_KEY}&redirect_uri=https://carreirausa.sigmaintel.io/api/docusign/callback
   ```
2. Visit URL in browser while logged in as admin
3. Review permissions requested (signature impersonation scope)
4. Click **Allow Access** to grant consent
5. You'll be redirected to callback URL (may show 404 - that's OK, consent is granted)

**Step 7: Configure Webhook (DocuSign Connect)**
1. Navigate to **Settings** → **Connect** → **Add Configuration**
2. Configuration Name: "Carreira AI Hub Webhooks"
3. Webhook URL: `https://carreirausa.sigmaintel.io/api/webhooks/docusign`
4. Enable **Include HMAC Signature**
5. Generate and copy HMAC secret (DOCUSIGN_WEBHOOK_SECRET)
6. Enable events:
   - ✓ Envelope Sent
   - ✓ Envelope Delivered
   - ✓ Envelope Completed
   - ✓ Envelope Declined
   - ✓ Envelope Voided
7. Save configuration

**Step 8: Prepare Credentials for Vercel**
Create a secure note with all values (you'll paste into Vercel in next task):

```
DOCUSIGN_INTEGRATION_KEY={Integration Key from Step 2}
DOCUSIGN_USER_ID={User ID from Step 4}
DOCUSIGN_ACCOUNT_ID={Account ID from Step 4}
DOCUSIGN_PRIVATE_KEY={Private key from Step 3 - multi-line or base64}
DOCUSIGN_BASE_URL={Base URL from Step 5}
DOCUSIGN_WEBHOOK_SECRET={HMAC secret from Step 7}
```

**Optional: DocuSign Template (if using template-based contracts)**
1. Navigate to **Templates** → **Create Template**
2. Upload contract PDF or create from scratch
3. Add text tabs with exact labels:
   - `customer_name`
   - `customer_email`
   - `invoice_number`
   - `amount`
   - `due_date`
   - `service_description`
4. Add signature and date tabs
5. Define role as "Client"
6. Save template and copy Template ID from URL
7. Add to credentials note: `DOCUSIGN_TEMPLATE_ID={Template ID}`
  </instructions>

<task type="checkpoint:human-action" gate="blocking">
  <action>Configure DocuSign credentials in Vercel production environment variables</action>
  <instructions>
**IMPORTANT:** This task requires access to Vercel dashboard and cannot be automated.

**Step 1: Access Vercel Project Settings**
1. Log in to https://vercel.com
2. Navigate to project: **carreirausa** (or current deployment project)
3. Click **Settings** → **Environment Variables**

**Step 2: Add DocuSign Environment Variables**

For each variable below, click **Add New** and enter:

**Required Variables:**

| Key | Value | Environment |
|-----|-------|-------------|
| `DOCUSIGN_INTEGRATION_KEY` | {Integration Key from DocuSign} | Production |
| `DOCUSIGN_USER_ID` | {User ID from DocuSign} | Production |
| `DOCUSIGN_ACCOUNT_ID` | {Account ID from DocuSign} | Production |
| `DOCUSIGN_PRIVATE_KEY` | {RSA Private Key - full PEM or base64} | Production |
| `DOCUSIGN_BASE_URL` | {https://na4.docusign.net or your region} | Production |
| `DOCUSIGN_WEBHOOK_SECRET` | {HMAC secret from Connect config} | Production |

**Optional Variables:**

| Key | Value | Environment |
|-----|-------|-------------|
| `DOCUSIGN_TEMPLATE_ID` | {Template ID if using templates} | Production |

**Important Notes:**
- **Environment scope:** Set all to **Production** only (not Preview or Development)
- **Private key format:** Can be multi-line PEM or base64 encoded single line
  - Multi-line: Paste entire key including `-----BEGIN RSA PRIVATE KEY-----` and `-----END RSA PRIVATE KEY-----`
  - Vercel handles multi-line values correctly
- **Security:** Never commit these values to git or share in public channels

**Step 3: Verify Variables Are Set**
1. After adding all variables, scroll through list
2. Confirm all 6 required variables show "Production" scope
3. Confirm DOCUSIGN_TEMPLATE_ID shows if you're using templates
4. Click **Save** if needed

**Step 4: Redeploy Production**
1. Navigate to **Deployments** tab
2. Find latest production deployment
3. Click **...** (three dots) → **Redeploy**
4. Select **Use existing Build Cache** (faster)
5. Click **Redeploy**
6. Wait for deployment to complete (~30-60 seconds)

**Step 5: Verify Environment Variables Are Loaded**
1. After deployment completes, note the deployment URL
2. You'll verify in next plan that DocuSign service can read these variables
  </instructions>
  <resume-signal>
Type "vercel-configured" when all environment variables are set in Vercel and production has been redeployed.

Provide deployment URL to confirm: e.g., "vercel-configured https://carreirausa-abc123.vercel.app"
  </resume-signal>
</task>
