# Phase 4.1: User Deployment - Dashboard & QuickBooks Data Validation - Context

**Gathered:** 2026-01-12
**Status:** Ready for planning

<vision>
## How This Should Work

The dashboard needs to be completely ready for real users across three departments: Commercial, Admin, and Finance. Each department should see a role-based view tailored to their needs - not a one-size-fits-all interface.

**Commercial Department** gets a comprehensive sales workspace:
- Sales pipeline visibility (leads, deals, qualification status)
- Lead activity tracking (conversations, AI scores, next actions)
- Performance metrics (conversion rates, deal velocity, revenue)
- Invoice creation capability (with Finance approval workflow)

**Finance Department** has dual responsibilities:
- Invoice approval workflow - review and approve/reject invoices created by Commercial
- QuickBooks data monitoring - ensure all financial data (customers, invoices, payments) is syncing correctly and completely

**Admin Department** gets full system control:
- System configuration (manage all integrations, OAuth tokens, settings)
- User management (create users, assign roles, manage permissions)
- Monitoring and logs (system health, integration logs, webhook status)

The experience should feel clean and minimal - like Linear or Stripe dashboards - not cluttered or overwhelming. Clear, modern interface with good use of white space and typography.

</vision>

<essential>
## What Must Be Nailed

All three of these are equally critical for deployment readiness:

- **Invoice approval workflow** - The handoff between Commercial creating invoices and Finance approving them must be smooth and reliable. This is a core business process.

- **QuickBooks data completeness** - Every required field must be captured and displayed correctly. No missing data, no incomplete syncs. Finance needs to trust the data.

- **Role-based access** - Each department must see only their tools and data. Security through proper role-based views, and clarity through focused interfaces.

</essential>

<boundaries>
## What's Out of Scope

- **No new integrations** - Stick with existing Pipedrive, QuickBooks, Stripe, DocuSign, Twilio. Don't add new external services.

- **No advanced analytics/reporting** - Just operational views needed for daily work. No BI dashboards or complex reports.

- **No mobile optimization** - Desktop first. Mobile experience can come later.

- **No new features** - Only deployment-critical improvements. Don't add new capabilities, just make existing features ready for users.

</boundaries>

<specifics>
## Specific Ideas

- **Design inspiration**: Linear and Stripe dashboards - clean, minimal, modern
- **Layout approach**: Role-based views, not a unified view for everyone
- **Commercial workflow**: Full sales pipeline + invoice creation capability
- **Finance workflow**: Dual focus on invoice approval AND QuickBooks monitoring
- **Admin access**: Complete system controls - configuration, users, monitoring, logs

</specifics>

<notes>
## Additional Context

This is the final deployment-readiness phase before users start working in the system. The technical foundation is solid (Phases 1-4 complete with webhook reliability, integration resilience, queue processing, and auth). Now it's about making the interface actually usable for real business operations.

The invoice approval workflow between Commercial and Finance is particularly important - this is a critical business handoff that must be reliable and clear.

QuickBooks data completeness is not just about syncing - it's about Finance being able to trust the data they see in the dashboard matches what's in QuickBooks.

</notes>
