1. Create `app/api/ops/daily/route.ts`:
- Import `getServerSession`, `authOptions`, `prisma`, `NextResponse`
- Import `SLA_DAYS_PER_PHASE`, `SLA_WARNING_DAYS`, `NO_SESSION_THRESHOLD_DAYS` from `@/lib/constants/sla`
- Auth guard: 401 if no session, 403 if role not ADMIN/OPERATIONAL
- Helper function:
```typescript
function daysBetween(a: Date, b: Date): number {
  return Math.floor((b.getTime() - a.getTime()) / 86_400_000);
}
```
- Query `prisma.mentorshipEnrollment.findMany`:
  - `where: { status: "ACTIVE", ...(role !== "ADMIN" ? { assignedToId: userId } : {}) }`
  - `include`: customer (id, name, qbBalance), currentPhase (id, label, key), assignedTo (id, name), transitions (orderBy createdAt desc, take 1, select createdAt), sessions (orderBy sessionDate desc, take 1, select sessionDate)
- For each enrollment compute:
  - `latestTransitionDate = enrollment.transitions[0]?.createdAt ?? enrollment.startDate`
  - `phaseAgeDays = daysBetween(latestTransitionDate, now)`
  - `daysRemaining = SLA_DAYS_PER_PHASE - phaseAgeDays`
  - `slaExpiring = daysRemaining <= SLA_WARNING_DAYS` (includes overdue where daysRemaining < 0)
  - `lastSessionDate = enrollment.sessions[0]?.sessionDate`
  - `daysSinceSession = lastSessionDate ? daysBetween(lastSessionDate, now) : Infinity`
  - `noRecentSession = daysSinceSession >= NO_SESSION_THRESHOLD_DAYS`
- Filter to only enrollments where `slaExpiring || noRecentSession`
- Map to response shape:
```typescript
{
  enrollmentId: string,
  studentName: string,
  customerId: string,
  phaseLabel: string,
  assigneeName: string,
  flags: { type: "sla_expiring" | "no_recent_session", daysRemaining?: number, daysSinceSession?: number }[],
}
```
- Sort: SLA-flagged first (more urgent), then by daysRemaining ascending
- Return `NextResponse.json({ students: [...], count: N })`
