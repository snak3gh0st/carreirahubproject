## Typography

| Role | Size | Weight | Line Height | Class |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-sans` |
| Label / badge | 11px | 600 (semibold) | 1.4 | `text-[11px] font-semibold` |
| Subheading | 14px | 600 (semibold) | 1.4 | `text-sm font-display font-semibold` |
| Heading / page title | 24px | 700 (bold) | 1.2 | `text-2xl font-display font-bold` |

Notes:
- Display headings use `font-display` (Blaak, serif).
- Body and UI labels use `font-sans` (Neue Montreal) or default sans; `font-display` is used for subheadings inside cards per existing ops pattern.
- Only 2 weights in use: regular 400 and bold/semibold 600–700 — consistent with ops hub pattern.

Source: Pre-populated from `app/ops/page.tsx` (`text-2xl font-display font-bold`, `text-sm`, `text-xs`) and `StudentCard.tsx` (`text-[13px]`, `text-[11px] font-mono`, `text-[10px] font-bold`).

## Color

| Role | Value | Tailwind Class | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | #FFFFFF | `bg-white` | Page background, card surfaces |
| Secondary (30%) | #2F443F (Verde) | `bg-brand-verde` | Sidebar, heading text, primary stats |
| Accent (10%) | #FF8142 (Tangerina) | `bg-brand-tangerina` / `text-brand-tangerina` | Active nav item only; used on dark (verde) bg |
| Flag: SLA expiring | #EF4444 (Red-500) | `bg-red-50 text-red-600 border-red-200` | SLA expiring badge and row accent |
| Flag: No recent session | #F59E0B (Amber-400) | `bg-amber-50 text-amber-600 border-amber-200` | No-session badge and row accent |
| Destructive | #EF4444 | `text-red-600` | No destructive actions in this phase |
| Success / empty state | #22C55E (Green-500) | `bg-green-50 text-green-700 border-green-200` | "Tudo certo hoje!" empty state banner |

Accent reserved for: active sidebar nav item background ONLY (Tangerina on Verde surface — never on white or Creme). All other interactive states use `text-brand-verde` or `hover:text-brand-verde`.

Tangerina on white/Creme is forbidden: contrast ratio ~2.9:1, fails WCAG AA. Source: brand identity memory + STATE.md decision.

Flag colors (red/amber) are semantic overrides for the flagging system — pre-populated from `StudentCard.tsx` `slaColor` pattern (`bg-red-500`, `bg-amber-400`).

## Component Inventory

### Flagged Student Row

Used in both `/ops/daily` and the flagged list section of `/ops/coordinator`.

Structure:
```
[flag badge(s)] [student name → profile link] [phase label] [days remaining/overdue] [→ profile link]
```

- Container: `bg-white rounded-xl border border-gray-200 px-4 py-3` (matches card pattern)
- Student name: `font-semibold text-sm text-gray-900 hover:text-brand-verde transition-colors` — wraps as `<Link href="/ops/students/{enrollmentId}">`
- Phase label: `text-xs text-gray-500`
- Days counter: `text-[11px] font-mono font-medium` — red if overdue, amber if warning
- SLA badge: `text-[10px] font-bold bg-red-50 text-red-500 border border-red-200 px-1.5 py-0.5 rounded-full uppercase tracking-wide` — label: "SLA"
- No-session badge: same shape as SLA badge, amber colors — label: "Sem sessão"
- When both flags active: show both badges side by side; SLA badge renders first (left) as higher urgency
- Direct profile link chevron: `text-brand-verde` lucide `ChevronRight` icon, 16px, `flex-shrink-0`

### Header Count Banner

Top of both pages above the flagged list.

- Container: `bg-white rounded-2xl border border-gray-200 p-6 mb-6`
- Text: `"X alunos precisam de atenção"` — `text-3xl font-bold text-brand-verde` for the number; `text-sm text-gray-500 mt-1` for the label
- When count = 0: replace with green success banner — see Empty State below

### Empty State Banner (Daily view, no flags)

- Container: `bg-green-50 rounded-2xl border border-green-200 p-8 text-center`
- Icon: lucide `CheckCircle2`, 32px, `text-green-600 mx-auto mb-3`
- Heading: `"Tudo certo hoje!"` — `text-lg font-display font-semibold text-green-700`
- Body: `"Nenhum aluno precisa de atenção no momento."` — `text-sm text-green-600`

### Phase Distribution Table (Coordinator page)

Reuse exact pattern from `app/ops/page.tsx` `withStudents.map(...)` section:

- Container: `bg-white rounded-2xl border border-gray-200 overflow-hidden`
- Header row: `px-6 py-4 border-b border-gray-100` with `font-display font-semibold text-brand-verde text-sm`
- Data rows: `px-6 py-3 flex items-center justify-between divide-y divide-gray-50`
- Phase label: `text-sm text-gray-700`
- Count: `text-sm font-semibold text-brand-verde`

Source: Directly reuse from `app/ops/page.tsx` lines 75–96.

### Debtors List (Coordinator page)

Same structure as Flagged Student Row but badge = "Débito" in red (matching pipeline `StudentCard` existing pattern).

### Sidebar Nav Additions

New entries added to `components/ops/ops-sidebar.tsx` `navItems` array:

| Entry | Label | Icon | Role Visibility |
|-------|-------|------|----------------|
| `/ops/daily` | `"Ações do Dia"` | `CalendarCheck` (lucide) | ADMIN + OPERATIONAL |
| `/ops/coordinator` | `"Coordenador"` | `LayoutList` (lucide) | ADMIN only |

- ADMIN-only entry: rendered conditionally — sidebar receives `userRole` prop; if `userRole !== "ADMIN"`, the coordinator item is not rendered.
- Active state: same `bg-brand-tangerina text-white font-semibold shadow-lg` class as existing items.

## Page Layouts

### `/ops/daily` — Ações do Dia

```
<div class="p-8">
  <h1>Ações do Dia</h1>          ← text-2xl font-display font-bold text-brand-verde
  <p>Bem-vindo, {name}</p>       ← text-sm text-gray-500 mt-1

  [Header Count Banner]          ← "X alunos precisam de atenção" OR green success banner

  [Flagged Student Row list]     ← divide-y divide-gray-100 inside rounded-2xl card
</div>
```

ADMIN sees unfiltered list (all assignees). OPERATIONAL sees only their assigned students.
No loading skeleton required — Server Component data fetch; use React Suspense fallback with `animate-pulse` gray placeholder rows if needed.

### `/ops/coordinator` — Visão do Coordenador

```
<div class="p-8">
  <h1>Visão do Coordenador</h1>  ← text-2xl font-display font-bold text-brand-verde

  [Header Count Banner]          ← "X alunos precisam de atenção hoje"

  [Flagged Student Row list]     ← all flagged students, no assignee filter

  <h2>Distribuição por Fase</h2> ← text-sm font-display font-semibold text-brand-verde, mt-10 mb-4
  [Phase Distribution Table]     ← React Query refetchInterval: 60_000

  <h2>Alunos com Débito QB</h2>  ← same subheading style, mt-8 mb-4
  [Debtors List]
</div>
```

ADMIN-only. Server-side redirect to `/ops` if role is not ADMIN.
Phase distribution table wrapped in `useQuery` client component for polling. Flagged list and debtors list are also in the polling scope (same query key).
