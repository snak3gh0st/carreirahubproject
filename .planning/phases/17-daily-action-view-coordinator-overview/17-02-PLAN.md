Three parallel queries via `Promise.all`:

**Query 1 — Phase distribution:**
```typescript
prisma.mentorshipPhase.findMany({
  orderBy: { sortOrder: "asc" },
  select: {
    label: true,
    key: true,
    _count: { select: { enrollments: { where: { status: "ACTIVE" } } } },
  },
})
```

**Query 2 — All active enrollments (for flags + no-session + debtors):**
```typescript
prisma.mentorshipEnrollment.findMany({
  where: { status: "ACTIVE" },
  include: {
    customer: { select: { id: true, name: true, qbBalance: true } },
    currentPhase: { select: { id: true, label: true, key: true } },
    assignedTo: { select: { id: true, name: true } },
    transitions: { orderBy: { createdAt: "desc" }, take: 1, select: { createdAt: true } },
    sessions: { orderBy: { sessionDate: "desc" }, take: 1, select: { sessionDate: true } },
  },
})
```

Process enrollments to produce three lists:

1. **flaggedStudents**: Same flag logic as daily route (SLA expiring OR no recent session), but NO assignee filter (COORD-02). Map to same shape as daily API response items.

2. **noSessionStudents**: Enrollments where `daysSinceSession >= NO_SESSION_THRESHOLD_DAYS`. Map to: `{ enrollmentId, studentName, phaseLabel, assigneeName, daysSinceSession }` (COORD-03).

3. **debtors**: Enrollments where `customer.qbBalance > 0`. Map to: `{ enrollmentId, studentName, phaseLabel, assigneeName, qbBalance }` (COORD-04).

Response shape:
```typescript
NextResponse.json({
  phaseDistribution: phases.map(p => ({ label: p.label, key: p.key, count: p._count.enrollments })),
  flaggedStudents: [...],
  flaggedCount: number,
  noSessionStudents: [...],
  debtors: [...],
})
```
