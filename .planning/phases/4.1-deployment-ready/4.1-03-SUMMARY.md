## Decisions Made

**Mobile Filter Approach:**
- Chose modal/drawer pattern over inline collapse for mobile (<768px)
- Rationale: Better UX on small screens, avoids pushing content down, full-screen focus
- Modal slides up from bottom with backdrop overlay
- Fixed Apply/Clear buttons at bottom for easy thumb access

**Table Responsiveness Strategy:**
- Horizontal scroll with hidden scrollbars instead of card view
- Rationale: Maintains consistency with desktop, users familiar with table UX
- Hidden non-critical columns on mobile (Source, Approval, Phone)
- Priority columns always visible: Customer/Name, Amount/Value, Date, Status

**Native Mobile Inputs:**
- Used type="date" for date inputs (triggers native date picker)
- Used inputMode="numeric" for number inputs (shows numeric keyboard)
- Rationale: Better mobile UX than custom pickers, accessibility built-in

**Touch Target Standards:**
- Minimum 44x44px for all interactive elements
- Rationale: Apple HIG standard, ensures accessibility and ease of use
- Applied to buttons, links, filter chips, table rows

**Column Hiding Strategy:**
- Invoices mobile: Show Invoice#, Customer, Amount, Due Date, Status (hide Source, Approval)
- Customers mobile: Show Name, Email, Balance, Status (hide Phone, Source)
- Payments mobile: Show Date, Customer, Amount, Status (hide Payment Method details)
- Rationale: Prioritize actionable data, reduce horizontal scroll distance
