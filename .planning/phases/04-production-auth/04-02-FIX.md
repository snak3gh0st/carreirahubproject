3. Check authentication middleware (middleware.ts):
   - Verify NextAuth session is properly configured
   - Check if auth middleware is unexpectedly blocking dashboard requests
   - Verify NEXTAUTH_SECRET is set in Vercel environment
4. Check dashboard layout/page files:
   - Look for async operations that might be throwing errors
   - Check for missing data fetches or database queries
   - Verify imports are working (no missing @/ path aliases)
5. Verify recent changes haven't broken dashboard:
   - Review commits from phase 04-01 and 04-02 that touch authentication
   - Check if token refresh logic inadvertently affects dashboard rendering

**Why this approach:**
- Vercel function logs have actual error message (only way to diagnose 500 errors)
- Database connectivity is fundamental - if schema not migrated, all queries fail
- Auth middleware could be unexpectedly blocking requests
- Phase 04 changes to auth could have side effects on dashboard
- Build succeeds but runtime 500 means code issue, not TypeScript issue
