## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`) | Icon gaps, scale button gaps in NPS row |
| sm | 8px (`gap-2`, `p-2`) | Compact form element padding, label-to-input gap |
| md | 16px (`gap-4`, `p-4`) | Default form field spacing, card inner padding |
| lg | 24px (`gap-6`, `p-6`) | Section padding (Forms panel card in student profile) |
| xl | 32px (`p-8`) | Page-level container padding (matches existing `p-8` in StudentProfileClient) |
| 2xl | 48px | Major section breaks between Forms section and Sessions section |
| 3xl | 64px | Not used in this phase |

Exceptions:
- NPS scale buttons: 44px min touch target height on mobile (`min-h-[44px]`) — accessibility requirement for tap targets.
- Scale button row: buttons `w-10 h-10` (40px) desktop, expand to `min-h-[44px]` on mobile.

Source: default 8-point scale. Touch target exception is WCAG 2.5.5.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFF8E8` (`--brand-creme` / `bg-white` for cards) | Page background, form card surfaces |
| Secondary (30%) | `#2F443F` (`--brand-verde`) | Section headings text, form label text, sidebar nav (inherited), scale number text |
| Accent (10%) | `#FF8142` (`--brand-tangerina`) | Selected NPS scale button highlight, "Assign Form" primary CTA button |
| Destructive | `hsl(var(--destructive))` (existing token) | Not used in this phase — no destructive actions |

Accent reserved for:
1. The currently selected scale button in the NPS 0-10 row (filled background, white text)
2. The "Assign Form" button in the ops FormsSection (primary CTA, matches ops portal CTA pattern)
3. Active/hover state on scale buttons (border-color only, not filled, for hover state)

Tangerina (#FF8142) MUST NOT be used as text on Creme or white backgrounds — fails WCAG AA (~2.9:1 contrast). It is valid only as: background fill with white text on top, or decorative border/icon on dark (Verde) surfaces.

NPS score badge in student profile: use Verde background (`bg-brand-verde`) with Creme text (`text-brand-creme`) — not Tangerina on white.

Unselected scale buttons: `bg-white border border-gray-200 text-gray-700` — neutral, not branded.
Completed form status badge: `bg-green-100 text-green-700` — using existing success semantic pattern.
Pending form status badge: `bg-yellow-100 text-yellow-700` — using existing warning semantic pattern.
Assigned (not started) form status badge: `bg-gray-100 text-gray-600`.

Source: brand.css, semantic.css, STATE.md WCAG constraint note for Tangerina.

## Visuals

Primary focal point: the "Atribuir Formulário" Tangerina-filled button is the visual anchor of the FormsSection card. It carries the only accent-color fill in the section and draws the operator's eye to the single available action.

Secondary focal point: the Verde section heading "Formulários" at the top of the FormsSection card. Verde at `text-base font-semibold` is the strongest text weight in the section and establishes the section boundary within the student profile.

Tertiary focal point: the form row status badges (Pendente / Em Andamento / Concluído). Their background fills (yellow, gray, green) are the only non-neutral inline color in the rows and communicate completion state at a glance without requiring the operator to read the row text.

## Interaction Contracts

### NPS Scale Field (in Hub `/hub/forms/[id]`)

The existing `scale` FieldType renders already. No new UI component needed. Visual contract for the existing renderer:
- 11 buttons (0-10) rendered in a single horizontal row, wrapping on narrow viewports
- Each button: `w-10 h-10 rounded-full border border-gray-200 text-sm font-semibold`
- Selected state: `bg-brand-tangerina text-white border-brand-tangerina` (Tangerina fill — white text passes WCAG on Tangerina background at ~4.5:1)
- Hover (unselected): `border-brand-verde text-brand-verde`
- Scale endpoints below the row: "0 — Pouco provável" (left) and "10 — Muito provável" (right) in `text-xs text-gray-500`
- The existing form renderer must not be modified structurally — only ensure these visual tokens are applied if the renderer uses hardcoded hex values

### Assign Form Dropdown (FormsSection in ops)

- Template dropdown: `<select>` element with Tailwind `block w-full rounded-md border border-gray-200 bg-white py-2 px-3 text-sm`
- Submit button: `bg-brand-tangerina hover:bg-brand-tangerina-500 text-white rounded-md px-4 py-2 text-sm font-semibold` — matches ops CTA button pattern
- Inline in the FormsSection card — no modal or drawer needed
- Show dropdown only when user clicks "Atribuir formulário" text button — collapse after submission

### NPS Score Display (in StudentProfileClient)

- Show only if a completed NPS submission exists for this enrollment's customer
- Inline badge: `inline-flex items-center gap-1 rounded-full bg-brand-verde px-3 py-1 text-xs font-semibold text-brand-creme`
- Content: "NPS Entrada: 8/10" or "NPS Saída: 7/10"
- Render one badge per NPS template (entry and exit separately)
- Position: inside the header card of StudentProfileClient, below the CEFR level badge (matches existing info cluster)

### Loading States

- FormsSection loading: render 3 skeleton rows (`animate-pulse bg-gray-100 h-12 rounded-lg`) — matches SessionSection loading pattern
- Assign action submitting: disable the submit button, show spinner icon from lucide-react (`Loader2 className="animate-spin"`)

### Empty States

- No forms assigned: render inside the FormsSection card:
  - Icon: `FileText` from lucide-react, `h-8 w-8 text-gray-300`
  - Heading: "Nenhum formulário atribuído" (`text-sm font-semibold text-gray-500`)
  - Body: "Atribua um formulário de entrada ou NPS para este aluno." (`text-xs text-gray-400`)

## Accessibility Constraints

- NPS scale buttons (0-10): each button must have `aria-label="Score {N}"` and `aria-pressed={selected}` — keyboard navigation must advance through the row with arrow keys (existing scale renderer should be verified)
- Forms status badges: use `role="status"` and sufficient color contrast — `bg-green-100 text-green-700` (4.5:1 verified), `bg-yellow-100 text-yellow-700` (4.5:1 verified)
- Tangerina on white: never used for text — all instances are filled button backgrounds with white foreground text
- Touch targets for assign button and form row action links: minimum 44px height
