# Plan 16-03: Human Verification Checkpoint

## Goal
Verify that the student profile page works end-to-end in a real browser session with live DB data.

## Automated checks (run before marking complete)

```bash
npx tsc --noEmit 2>&1 | grep -E "students|SessionSection|StudentProfile"
```

Expected: no output (zero errors in phase 16 files).

## Human verification items

### 1. Profile page renders from pipeline card
**Test:** Start `npm run dev`, navigate to `/ops/pipeline`. Click a student name on a card.
**Expected:** Browser navigates to `/ops/students/[enrollmentId]` and the profile page loads with the student's name, email, program badge, current phase, and assigned team member in the header section.
**Why human:** Server-side data fetch and client navigation require a real browser session.

### 2. CEFR result appears when test exists
**Test:** Click a student who has completed the placement test.
**Expected:** An "Inglês (CEFR)" row appears in the header with the display level (e.g. "B2") and percentage.
**Why human:** Requires a student record with a PlacementTest row linked in the DB.

### 3. Phase timeline renders in chronological order
**Test:** Open a profile for a student who has advanced at least one phase.
**Expected:** The "Histórico de Fases" section shows a vertical timeline. Each entry shows "Início → [Phase]" or "[Phase] → [Phase]", the date/time, and the name of the user who triggered the transition.
**Why human:** Requires real PhaseTransition rows in the DB.

### 4. Log session form submits without page reload
**Test:** Click "Registrar Sessão", fill in type, conductor, date. Click "Salvar".
**Expected:** The form disappears, a success toast appears ("Sessão registrada com sucesso"), and the new session appears at the top of the session list — all without a full page reload.
**Why human:** Form submission and optimistic UI require browser interaction.

### 5. Session pagination works
**Test:** Open a profile with more than 20 sessions (or manually add sessions to exceed the threshold).
**Expected:** Pagination controls "Anterior" / "Próximo" appear. Clicking "Próximo" loads the next 20 sessions.
**Why human:** Requires sufficient session data in the DB.
