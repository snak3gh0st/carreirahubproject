Create `lib/services/mentorship.service.ts` following the stateless singleton pattern from `lib/services/invoice-workflow.service.ts` (class + `export const mentorshipService = new MentorshipService()` at the bottom).

The service must implement exactly these three public methods:

---

**`createEnrollment(data)`** — per D-07 and D-08:

Input type:
```typescript
{
  customerId: string;
  programType: "PASS" | "ADVANCED";
  assignedToId: string;
  startDate: Date;
  triggeredById: string; // the User performing the enrollment
}
```

Logic:
1. Check for existing ACTIVE enrollment: `prisma.mentorshipEnrollment.findFirst({ where: { customerId, status: "ACTIVE" } })`. If found, throw an error with code `"DUPLICATE_ENROLLMENT"` and message `"This student is already enrolled in an active ${existing.programType} program."` — the API route will convert this to 409.
2. Find the bastao phase: `prisma.mentorshipPhase.findUniqueOrThrow({ where: { key: "bastao" } })`.
3. Use `prisma.$transaction(async (tx) => { ... })` (interactive transaction variant) to:
   a. Create the `MentorshipEnrollment` with `status: "ACTIVE"`, `currentPhaseId: bastaoPhase.id`.
   b. Create the `PhaseTransition` with `enrollmentId: enrollment.id`, `fromPhaseId: null`, `toPhaseId: bastaoPhase.id`, `triggeredById`.
4. Return `{ enrollment, transition }`.

---

**`logSession(data)`** — per D-12 and D-13:

Input type:
```typescript
{
  enrollmentId: string;
  sessionType: string;
  conductorId: string;
  sessionDate: Date;
  notes?: string;
}
```

Logic:
1. Verify the enrollment exists and is ACTIVE: `prisma.mentorshipEnrollment.findFirstOrThrow({ where: { id: enrollmentId, status: "ACTIVE" } })`. Throw a plain Error with message `"Enrollment not found or not active"` if missing (Prisma will throw P2025 — catch it and re-throw with this message).
2. Create the session: `prisma.mentorshipSession.create({ data: { enrollmentId, sessionType, conductorId, sessionDate, notes } })`.
3. Return the created session.

---

**`advancePhase(data)`** — per D-09, D-10, D-11:

Input type:
```typescript
{
  enrollmentId: string;
  toPhaseId: string;
  triggeredById: string;
  triggeredByRole: string; // "ADMIN" | "OPERATIONAL" | ...
}
```

Logic:
1. Fetch the enrollment with `currentPhase`: `prisma.mentorshipEnrollment.findFirstOrThrow({ where: { id: enrollmentId, status: "ACTIVE" }, include: { currentPhase: true } })`.
2. Fetch `toPhase`: `prisma.mentorshipPhase.findUniqueOrThrow({ where: { id: toPhaseId } })`.
3. Forward-only enforcement for non-ADMIN (per D-09): if `triggeredByRole !== "ADMIN"`, verify `toPhase.sortOrder === (currentPhase.sortOrder + 1)`. If not, throw Error with code `"INVALID_TRANSITION"` and message `"Phase transitions must be sequential. Expected sortOrder ${currentPhase.sortOrder + 1}, got ${toPhase.sortOrder}."`.
4. Use `prisma.$transaction(async (tx) => { ... })` to:
   a. Create `PhaseTransition` with `fromPhaseId: enrollment.currentPhaseId`, `toPhaseId`, `triggeredById`.
   b. Update `MentorshipEnrollment` setting `currentPhaseId: toPhaseId`.
5. Return `{ transition, enrollment }`.
