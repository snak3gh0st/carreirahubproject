<task type="auto">
  <name>Task 1: Implement Payment Intent creation API endpoint</name>
  <files>app/api/stripe/payment-intent/route.ts, lib/services/stripe.service.ts</files>
  <action>
    Create API endpoint to generate Payment Intents for invoice payments.

    **API endpoint (app/api/stripe/payment-intent/route.ts):**
    - Create POST endpoint at `/api/stripe/payment-intent`
    - Accept JSON body: `{ invoiceId: string, savePaymentMethod?: boolean }`
    - Validate input with Zod schema
    - Fetch invoice from database with customer relationship
    - Call `stripeService.createPaymentIntentForInvoice()` (new method)
    - Return Payment Intent client_secret for frontend

    **New service method (lib/services/stripe.service.ts):**
    - Add `createPaymentIntentForInvoice(invoice: Invoice)` method
    - Get or create Stripe customer using existing `getOrCreateCustomer()` method
    - Update database customer with `stripe_id` if newly created
    - Create Payment Intent with:
      - customer: Stripe customer ID
      - amount: invoice amount * 100 (convert to cents)
      - currency: 'usd'
      - automatic_payment_methods: { enabled: true } (enables cards, US bank accounts, etc.)
      - metadata: { invoice_id, invoice_number, hub_customer_id }
      - setup_future_usage: 'off_session' (if savePaymentMethod=true, allows saving card for future)
    - Log to IntegrationLog with service="stripe", action="createPaymentIntent"
    - Return Payment Intent object

    **Error handling:**
    - Handle circuit breaker open (return 503 Service Unavailable)
    - Handle invalid invoice ID (return 404)
    - Handle Stripe API errors (log and return 500)
    - Log all errors to IntegrationLog

    **What to avoid:**
    - Do NOT use deprecated Charges API (use Payment Intents API - it supports 3D Secure/SCA)
    - Do NOT create invoice in Stripe here (that's for subscription billing in Plan 3)
    - Do NOT implement the frontend payment form (that's out of scope - API only)
  </action>
  <verify>
    1. Endpoint at `/api/stripe/payment-intent` accepts POST
    2. Returns Payment Intent with client_secret
    3. Stripe customer created if doesn't exist
    4. Database customer updated with stripe_id
    5. Payment Intent metadata includes invoice_id
    6. Test: curl -X POST /api/stripe/payment-intent -d '{"invoiceId":"xxx"}' returns client_secret
  </verify>
  <done>
    - API endpoint created and working
    - Payment Intent created in Stripe
    - Customer created/retrieved in Stripe
    - Database customer has stripe_id
    - Returns client_secret for frontend
    - All operations logged to IntegrationLog
  </done>
</task>

  <name>Task 2: Handle payment_intent.succeeded webhook for payment confirmation</name>
  <files>app/api/webhooks/stripe/route.ts, lib/services/stripe.service.ts</files>
  <action>
    Update webhook endpoint to process successful payments and update invoice status.

    **Webhook handler updates (app/api/webhooks/stripe/route.ts):**
    - In `payment_intent.succeeded` event handler:
      1. Extract Payment Intent ID from event
      2. Extract invoice_id from Payment Intent metadata
      3. Fetch invoice from database with customer relationship
      4. Validate invoice exists and is not already paid
      5. Call `stripeService.handlePaymentSuccess()` (new method)
      6. Log success to IntegrationLog
    - Use transaction to prevent race conditions
    - If invoice not found in metadata: log warning but return 200 (don't fail webhook)

    **New service method (lib/services/stripe.service.ts):**
    - Add `handlePaymentSuccess(paymentIntentId: string, invoice: Invoice)` method
    - Retrieve full Payment Intent from Stripe to get payment details
    - Update invoice in database:
      - status: 'PAID'
      - paidAt: Payment Intent created timestamp
      - amountPaid: Payment Intent amount / 100
      - paymentMethod: Payment Intent payment_method_types[0]
      - stripePaymentIntentId: Payment Intent ID
    - DO NOT sync to QuickBooks yet (that's Task 3)
    - Return updated invoice

    **What to avoid:**
    - Do NOT send email receipts here (email service not in scope for Sprint 1)
    - Do NOT handle refunds yet (that's for Plan 4 if needed)
    - Do NOT sync to QB yet (next task)
  </action>
  <verify>

    2. Invoice paidAt timestamp set correctly
    3. Invoice amountPaid matches Payment Intent amount
    4. stripePaymentIntentId stored in invoice
    5. Duplicate webhooks don't cause double-processing (deduplication working)
    6. Test: Trigger payment_intent.succeeded event, check invoice status updated
  </verify>
