  <name>Task 1: Enhance Identity Mapper for DocuSign integration and bidirectional sync</name>
  <files>
    lib/services/identity-mapper.ts,
    lib/services/docusign.service.ts,
    app/api/webhooks/docusign/route.ts,
    app/api/webhooks/quickbooks/route.ts
  </files>
  <action>
    Extend Identity Mapper to support DocuSign and implement bidirectional customer sync:

    1. **In lib/services/identity-mapper.ts**:
       - Add `docusign_id` to ExternalIds interface (should already exist in schema)
       - Update reconcileCustomer() to handle externalIds.docusign_id
       - Add method: syncToDocuSign(customerId) - Push customer data to DocuSign when updated
       - Add method: syncFromDocuSign(docusignId, customerData) - Pull customer updates from DocuSign
       - Add conflict resolution: Compare updatedAt timestamps, last-write-wins

    2. **In lib/services/docusign.service.ts**:
       - Add method: createOrUpdateContact(customerData) - Create/update DocuSign contact
       - Return DocuSign contact ID for storage in Customer.docusign_id
       - Handle API errors gracefully (log to IntegrationLog, return null on failure)

    3. **In app/api/webhooks/docusign/route.ts**:
       - On envelope-completed event: Extract signer email, reconcile customer via Identity Mapper
       - Update Customer.docusign_id if not already set
       - Log event to IntegrationLog

    4. **In app/api/webhooks/quickbooks/route.ts**:
       - On Customer.Update event: Reconcile customer via Identity Mapper
       - If customer has docusign_id, sync changes to DocuSign via identityMapper.syncToDocuSign()
       - Log sync to IntegrationLog

    5. **Customer sync flow**:
       - QuickBooks customer updated → Webhook → Identity Mapper reconcile → Sync to DocuSign (if exists)
       - DocuSign envelope signed → Webhook → Identity Mapper reconcile → Update Customer.docusign_id
       - Manual customer update (if needed) → Sync to both QuickBooks and DocuSign

    6. **What to avoid and WHY**:
       - Do NOT sync on every field change - only sync when name, email, or phone changes (avoid API spam)
       - Do NOT fail workflow if sync fails - log error and continue (eventual consistency is acceptable)
       - Do NOT create real-time sync - webhook-driven is sufficient (no polling needed)
       - Do NOT sync DocuSign → QuickBooks automatically - QuickBooks is source of truth for financial data
  </action>
